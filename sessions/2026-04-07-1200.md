### 2. Daily Hours Data Recovery (commit 4dbc473)
User reported all historical daily hours had disappeared.

**Root cause**: Migration 001 added `job_site_id` to `daily_hours` as nullable — all pre-existing records had `job_site_id = NULL`. The app query `.eq('job_site_id', currentJobSite.id)` never matches NULL. Data was never deleted — just invisible.

**Fix**: Migration 024 (`024_backfill_daily_hours_job_site_id.sql`):
- Step 1: Delete NULL records where a site-specific record already exists for same worker/date (prevents constraint violation)
- Step 2: Backfill `job_site_id = workers.job_site_id` on remaining NULL records

**Action required**: Run `migrations/024_backfill_daily_hours_job_site_id.sql` in Supabase SQL editor.
